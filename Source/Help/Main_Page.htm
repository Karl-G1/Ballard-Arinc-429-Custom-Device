Introduction

This document is the documentation for the Parameters XML files used in Ballard Arinc-429 Custom Device.

XML Schema

XML Schema used is called: Arinc429_Parameters_XML_Schema.xsd. Schema is located at location: …\NI VeriStand XXXX\Custom Devices\Ballard - Arinc-429\Data (XXXX being your NI-VeriStand version

Caution Do not edit the contents of the XML Schema.

For XML Schema File, go to XML Schema File.

XML file validation against Schema

The Ballard Arinc-429 Custom Device comes with an Assembly called: Parameters_Arinc429_XML_Tools.dll, located here: …\NI VeriStand XXXX\Custom Devices\Ballard - Arinc-429\Data (XXXX being your NI-VeriStand version). The assembly can be called to check any user defined XML Parameters file against the schema. Though, it's important to note that some extra checkngs are performed out from the Assembly itself.

Once the validation against XML Schema is done, some extra checking is performed. Below is a list of what is checked. Left Column gives the Error Code returned in case of Error. Right Column is the Description of the checking performed.

Error Description
5010 This checking confirms that, for a given Hardware Channel+Label combination, all parameters have a unique combination of Parameter settings: Hardware Channel, Label Decimal and Parameter Name. If there are multiple duplicates, the feature returns all duplicates.
5011 This checking confirms that, for a given Hardware Channel+Label combination, all parameters don't use a range of bits larger than 32. If there are some Parameters out of range, the feature returns their names.
5012 This checking confirms that, for a given Hardware Channel+Label combination, all Discrete parameters have a name per Bit. If there are some Discrete Parameters without name, the feature returns their names.
5013 This checking confirms that, for a given Tx Hardware Channel, all Parameters have a Default Value (per Bit for Discrete Parameter Type). If there are some Discrete Parameters without Default Value, the feature returns their names.
5014 This checking confirms that, for a given Hardware Channel+Label combination, all BCD parameters have a valid startBit+numberOfBits combination. If some BCD Parameters have invalid settings, the feature returns their names.
5015 This checking confirms that, for a given Tx Hardware Channel, all Labels have their createParityChannel set to false. If there are some Labels with createParityChannel set to true, the feature returns their names.
5016 This checking confirms that, for a given Tx Hardware Channel, all Parameters have their startBit set to a minimum value of 8. If there are some Parameters with startBit equal or lower than 7, the feature returns their names.
