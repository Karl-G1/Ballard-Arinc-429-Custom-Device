XML Parameters Files Examples

BNR Parameter Example

The table, below, is an example of a BNR Parameter.

Line XML Code
1. <label>
2.     <labelDecimal>23</labelDecimal
3.     <createSDIChannel>false</createSDIChannel
4.     <createSSMChannel>true</createSSMChannel
5.     <createParityChannel>true</createParityChannel
6.     <parameter>
7.         <encoding>BNR</encoding
8.         <signed>true</signed
9.         <startBit>10</startBit
10.         <numberOfBits>19</numberOfBits
11.         <scale>234.6</scale
12.         <offset>-2.4</offset
13.         <name>Parameter 0</name
14.         <unit>psi</unit
15.     </parameter>
16. </label>

NOTES:

  • This example doesn't show the source code for the <channel> and <hardwareChannel>;
  • It's a Parameter definition for a Rx Channel as <defaultValue> element is not present;
  • For a given <channel>, there can be multiple instances of <label>;
  • For a given <label>, there can be multiple instances of <parameter>;
  • The Engine takes 19 Bits, starting the 11th Bit (Bit #10) in the Raw Arinc-429 word. As it's a signed BNR Parameter, the most left Bit is the Sign Bit. The engine converts these 19 Bits (2's complement) to decimal, multiplies by the <scale> value and adds the <offset> value;
  • This Parameter will have a NI-VeriStand Channel named Parameter 0. Unit for that NI-VeriStand Channel will be psi.

BCD Parameter Example

Line XML Code

26.         </parameter>

28. <channel>

NOTES:

  • These are Parameters definition for a Tx Channel as <defaultValue> element is present and <hardwareChannel> element is in [16:31] range;
  • For a given <channel>, there can be multiple instances of <label>;
  • For a given <label>, there can be multiple instances of <parameter>;
  • Below is the breakdown of <labelDecimal>28 with individual Label Bits and BCD Digits (top row represents Bits Indexes):
  • This Parameter will have a NI-VeriStand Channel named "Parameter 5". Unit for that NI-VeriStand Channel will be: psi.
  • For Parameter Parameter 5, "SSM" Bits (Bits [29:30]) are used for the sign, we consider it negative if SSM Bits are 11b (0x3). For all other SSM Bits values, we consider the BCD to be positive. Bits [18:28] are 3 digits ("CHAR 1", "CHAR 2" and "CHAR 3"). As no Scaling is applied, value can vary in range: [-799:799]. Most left digit "CHAR 1" has only 3 bits width;
  • For Parameter Parameter 16, Bits [10:17] are 2 digits ("CHAR 4" and "CHAR 5"). As no Scaling is applied, value can vary in range: [0:99].

Discrete Parameter Example

The table, below, is an example of a Discrete Parameter.

Line XML Code
1. <label>
2.     <labelDecimal>23</labelDecimal
3.     <parameter>
4.         <encoding>Discrete</encoding
5.         <startBit>10</startBit
6.         <numberOfBits>1</numberOfBits
7.         <name>Parameter 30</name
8.     </parameter>
9. </label>

NOTES:

  • This example doesn't show the source code for the <channel> and <hardwareChannel>;
  • It's a Parameter definition for a Rx Channel as <defaultValue> element is not present;
  • For a given <channel>, there can be multiple instances of <label>;
  • For a given <label>, there can be multiple instances of <parameter>;
  • The Engine takes 1 Bit, starting the 11th Bit (Bit #10) in the Raw Arinc-429 word. No Scaling is applied;
  • This Parameter will have a NI-VeriStand Channel named Parameter 30.

Discrete Parameters Example

The table, below, is an example of Discrete Parameters for a Label from a Tx Channel.

Line XML Code
1. <label>
2.     <labelDecimal>26</labelDecimal
3.     <parameter>
4.         <encoding>Discrete</encoding
5.         <startBit>10</startBit
6.         <numberOfBits>3</numberOfBits
7.         <name>Parameter 30</name
7.         <name>Parameter 31</name
7.         <name>Parameter 32</name
25.         <defaultValue>1</defaultValue
25.         <defaultValue>0</defaultValue
25.         <defaultValue>1</defaultValue
8.     </parameter>
9. </label>

NOTES:

  • This example doesn't show the source code for the <channel> and <hardwareChannel>;
  • It's a Parameter definition for a Rx Channel as <defaultValue> element is not present;
  • For a given <channel>, there can be multiple instances of <label>;
  • For a given <label>, there can be multiple instances of <parameter>;
  • For Parameter Parameter 30, NI-VeriStand Engine sets Bit #10 (11th Bit) in the Raw Arinc-429 word. No Scaling is applied. Default Value is 1;
  • For Parameter Parameter 31, NI-VeriStand Engine sets Bit #11 (12th Bit) in the Raw Arinc-429 word. No Scaling is applied. Default Value is 0;
  • For Parameter Parameter 32, NI-VeriStand Engine sets Bit #12 (13th Bit) in the Raw Arinc-429 word. No Scaling is applied. Default Value is 1.
